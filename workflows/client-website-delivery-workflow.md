# Client Website Delivery Workflow

## Purpose
Create a repeatable process for turning client documentation into a live deployed website.

## Trigger
Use for client landing pages, service sites, portfolios, event sites, and MVP web apps.

## Inputs
- Goal, audience, and desired artifact
- Source material, constraints, risks, and approval status

## Step-by-Step Process
1. Receive client documentation
2. Store private client files outside the public repo
3. Extract requirements
4. Identify missing information
5. Create a website specification
6. Create an AI MVP build prompt
7. Build the MVP using the chosen AI coding tool
8. Review the MVP manually
9. Fix design, content, accessibility, and functionality issues
10. Initialize GitHub repo or connect existing repo
11. Push code to GitHub
12. Connect GitHub repo to Netlify
13. Configure build command and publish directory
14. Add environment variables in Netlify if needed
15. Deploy preview
16. Test live site
17. Share with client for review
18. Track client feedback
19. Apply revisions
20. Prepare handoff notes

## Output
A Markdown-ready deliverable with summary, decisions, assumptions, risks, quality checks, and next steps.

## Quality Control
Verify that the output is grounded in source context, privacy-safe, scoped to the goal, and clear enough for another person or agent to continue.

## Failure Modes
Missing source context, hidden assumptions, overbroad scope, unapproved external actions, weak privacy handling, and unclear ownership.

## Example
Use this workflow to move from a rough request to a reviewed, documented, and safe next-step artifact.
