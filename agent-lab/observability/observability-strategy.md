# Observability Strategy

## Why Agent Runs Need Logs
Logs make agent work reviewable, debuggable, and trustworthy. They help explain what context was used, which decisions were made, which tools were touched, and what follow-up remains.

## What To Capture
Capture date, agent, task, inputs, Integration Adapters used, output produced, approval gates, risks, decisions, errors, and follow-up actions.

## What Never To Log
Do not log secrets, credentials, raw private files, sensitive personal data, private client content, or unnecessary identifying details.

## How Observability Helps
Good logs make failures easier to reproduce, support version comparison, reveal risky patterns, and help stakeholders understand how an output was created.
