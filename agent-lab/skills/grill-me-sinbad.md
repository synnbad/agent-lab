# Grill-Me Sinbad

## Purpose
Stress-test ideas, plans, specs, technical designs, client website scopes, assessment plans, projects, app concepts, and major decisions before execution.

## When To Use
Use this skill when a plan feels important, expensive, risky, vague, or likely to create downstream rework.

## Inputs Needed
- Current idea, plan, spec, or decision
- Goal, constraints, audience, and timeline
- Any available codebase, artifact, or source context

## Procedure
1. Build a current understanding from the provided context.
2. Inspect available files or artifacts before asking questions that can be answered from context.
3. Identify assumptions, weak spots, missing constraints, vague goals, and hidden risks.
4. Ask sharp, non-obvious questions, one at a time when the topic is complex.
5. Resolve dependencies between decisions before final recommendations.
6. Give a recommended answer whenever a strong default is obvious.
7. Produce a refined plan and execution checklist.

## Output
1. Current Understanding
2. Critical Unknowns
3. Questions, one at a time if needed
4. Recommended Answers
5. Risk Map
6. Final Refined Plan
7. Execution Checklist

## Checks
- Questions materially improve the plan.
- Feedback is precise, direct, and useful without being performatively harsh.
- The final plan prevents bad execution rather than merely criticizing the idea.
