# Dashboard Storytelling

## Purpose
Turn dashboard metrics into a clear narrative for decisions and action.

## When To Use
Use this skill when a dashboard needs an executive explanation, walkthrough, or improvement plan.

## Inputs Needed
- Goal, audience, and decision the work supports
- Source material, constraints, and known limitations

## Procedure
1. Confirm the purpose, audience, and desired output.
2. Review the source material and identify gaps, risks, and assumptions.
3. Produce a structured result with evidence, limitations, and next steps.

## Output
A concise Markdown artifact with summary, source basis, analysis, recommendations, limitations, and follow-up actions.

## Checks
- The output is grounded in provided evidence.
- Assumptions, risks, and privacy constraints are clearly labeled.

## Example Usage
Use the Dashboard Storytelling skill to review this sanitized material and produce a practical next-step artifact.
