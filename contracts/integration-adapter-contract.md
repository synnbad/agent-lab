# Integration Adapter Contract

## Purpose
Adapter contract fields: adapter name, platform aliases, connected agents, access level, read/write/deploy capability, required permissions, approval needed, data sensitivity, safe usage pattern, failure fallback, and security risks.

## Required Fields
- Goal or artifact name
- Source basis
- Constraints
- Risk level
- Approval status

## Optional Fields
- Deadline or priority
- Owner
- Related files
- Follow-up review date

## Validation Checklist
- Required fields are complete
- Assumptions are labeled
- Sensitive data is excluded or approved
- Approval status is clear
- Output can be handed to another agent or reviewer

## Example
Use this contract to capture the minimum safe context before starting or transferring work.
