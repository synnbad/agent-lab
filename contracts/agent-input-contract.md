# Agent Input Contract

## Purpose
Minimum fields: task goal, user intent, active agent, context provided, source files or links, constraints, desired output, risk level, required integrations, approval status, open questions, and assumptions.

## Required Fields
- Goal or artifact name
- Source basis
- Constraints
- Risk level
- Approval status

## Optional Fields
- Deadline or priority
- Owner
- Related files
- Follow-up review date

## Validation Checklist
- Required fields are complete
- Assumptions are labeled
- Sensitive data is excluded or approved
- Approval status is clear
- Output can be handed to another agent or reviewer

## Example
Use this contract to capture the minimum safe context before starting or transferring work.
