# Contract Strategy

## Purpose
Contracts reduce ambiguity between agents, support cross-platform compatibility, improve handoffs, guide artifact reviews, document Integration Adapter use, and define acceptance criteria. Agents should not start complex work until minimum input requirements are met or assumptions are clearly labeled.

## Required Fields
- Goal or artifact name
- Source basis
- Constraints
- Risk level
- Approval status

## Optional Fields
- Deadline or priority
- Owner
- Related files
- Follow-up review date

## Validation Checklist
- Required fields are complete
- Assumptions are labeled
- Sensitive data is excluded or approved
- Approval status is clear
- Output can be handed to another agent or reviewer

## Example
Use this contract to capture the minimum safe context before starting or transferring work.
