# Artifact Review Contract

## Purpose
Artifacts include reports, website specs, MVP build prompts, dashboard requirements, client handoff notes, assessment plans, stakeholder briefs, GitHub documentation, and deployment checklists. Review criteria include accuracy, completeness, clarity, privacy, security, stakeholder readiness, technical feasibility, accessibility, versioning, and approval status.

## Required Fields
- Goal or artifact name
- Source basis
- Constraints
- Risk level
- Approval status

## Optional Fields
- Deadline or priority
- Owner
- Related files
- Follow-up review date

## Validation Checklist
- Required fields are complete
- Assumptions are labeled
- Sensitive data is excluded or approved
- Approval status is clear
- Output can be handed to another agent or reviewer

## Example
Use this contract to capture the minimum safe context before starting or transferring work.
