# Handoff Contract

## Purpose
Required handoff fields: source agent, target agent, reason for handoff, task goal, context summary, source files, decisions already made, constraints, risks, open questions, expected output from target agent, deadline or priority if relevant, and approval status.

## Required Fields
- Goal or artifact name
- Source basis
- Constraints
- Risk level
- Approval status

## Optional Fields
- Deadline or priority
- Owner
- Related files
- Follow-up review date

## Validation Checklist
- Required fields are complete
- Assumptions are labeled
- Sensitive data is excluded or approved
- Approval status is clear
- Output can be handed to another agent or reviewer

## Example
Use this contract to capture the minimum safe context before starting or transferring work.
