# Integration Terminology Glossary

## Integration Adapter
Any external system, app, connector, action, tool, function, API, MCP server, plugin, knowledge source, or deployment integration that lets an AI agent read context, retrieve data, take action, write files, create artifacts, or deploy work.

## Connector
A platform-specific connection to an external data source or app.

## App
A user-facing platform extension that may provide context, actions, or content creation.

## Action
A callable operation that performs a task, often through an API or workflow.

## Tool
A capability available to an AI agent, such as search, file access, shell execution, or API calls.

## Function Calling
A structured way for a model to request that software execute a defined function.

## API Action
A tool-backed operation that calls an external API.

## MCP Server
A Model Context Protocol server that exposes tools, resources, or prompts to compatible clients.

## MCP Tool
A callable capability exposed by an MCP server.

## MCP Resource
Reference content exposed by an MCP server for model context.

## MCP Prompt
A reusable prompt exposed by an MCP server.

## Plugin
An extension bundle that adds capabilities, integrations, or workflow support.

## Knowledge Source
A document store, index, folder, database, or content system used for retrieval.

## Agent Flow
A configured process that routes tasks, tools, and actions across an agent workflow.

## Context Provider
A system that supplies relevant context to the agent without necessarily allowing writes.

## Deployment Integration
An adapter used to build, deploy, publish, or inspect live systems.

## Read-only Integration
An adapter limited to inspection, search, retrieval, or summarization.

## Write Integration
An adapter that can create or modify artifacts, comments, files, issues, or external records.

## Destructive Action
An action that can delete, overwrite, publish, revoke, rotate, deploy, or otherwise change state in a way that may be hard to undo.

## Sensitive Data Source
Any source containing private institutional, client, student, staff, patron, HR, medical, financial, immigration, or personally identifiable data.
