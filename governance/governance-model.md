# Governance Model

Agent changes are owned by the repository maintainer. Changes should be reviewed for purpose, risk, guardrails, examples, Integration Adapter assumptions, contracts, and evaluation coverage. High-risk agents require explicit review before use on real client, deployment, sensitive-data, or stakeholder-facing work. Integration Adapters require review of permissions, access level, approval gates, and fallback instructions. Sensitive data use requires human approval and documented minimization.
